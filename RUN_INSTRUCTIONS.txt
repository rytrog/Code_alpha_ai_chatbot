══════════════════════════════════════════════════════════════
HOW THE 9-STAGE PIPELINE WORKS
══════════════════════════════════════════════════════════════

  User Question
       │
       ├─ 1. Is it a GREETING? (hello, hi, hey) → Instant reply
       ├─ 2. Normalize question → remove stopwords
       ├─ 3. Is it in FAQ database? → Database answer
       ├─ 4. Is it in Answer Cache? → Cached answer
       ├─ 5. Is it university-related? → Reject if off-topic
       ├─ 6. Search ChromaDB → Retrieve relevant chunks
       ├─ 7. Call Gemini 2.5 Flash → Generate answer from context
       ├─ 8. Save to cache → For future identical questions
       └─ 9. Return answer to user

  Stages 1-5: NO AI API call (saves costs)
  Stage 7: Only called when necessary

TROUBLESHOOTING
══════════════════════════════════════════════════════════════

  Problem: "ConnectionRefusedError" on startup
  Fix:     PostgreSQL is not running. Start it first.

  Problem: AI answers say "AI service is not configured"
  Fix:     Set GEMINI_API_KEY in your .env file.

  Problem: "No documents ingested" warnings
  Fix:     Upload PDF/TXT/CSV files via the admin upload page.

  Problem: Out-of-scope for valid university questions
  Fix:     The scope checker uses keywords. If your question
           doesn't contain a university keyword, it's rejected.
           Add keywords to backend/data/university_keywords.json

  Problem: Port 8000 already in use
  Fix:     Change PORT in .env or kill the existing process:
           netstat -ano | findstr :8000
           taskkill /PID <pid> /F

embeding in a web :
To embed the chatbot on any other website (like the main university website or a department page), the teacher or HOD only needs to add a single script tag near the bottom of their website's HTML file (just before the closing </body> tag).

Here is the exact code block you should give them:
<!-- AITD Kanpur AI Chatbot Widget -->
<script 
  src="http://YOUR_SERVER_IP:8000/widget/widget.js" 
  data-api="http://YOUR_SERVER_IP:8000/api/chat">
</script>
Features:
Zero Configuration: The widget is fully self-contained. It automatically loads its own styles, HTML structure, and interactive icon in the bottom-right corner of the page.
CORS Pre-Configured: The backend in app.py has Cross-Origin Resource Sharing (CORS) enabled for all origins ("*"), meaning external websites can make API requests to it without any blocking issues.


How to Install It on Different Platforms:
1. Plain HTML Website
Paste the script tag directly into your main HTML file (usually index.html), just before the </body> tag:

html
...
  <!-- Chatbot Widget -->
  <script src="http://YOUR_SERVER_IP:8000/widget/widget.js" data-api="http://YOUR_SERVER_IP:8000/api/chat"></script>
</body>
</html>

1. Google Gemini (Google AI Studio) — Cheapest & Best for RAG
Google's Gemini 1.5 Flash model has a massive context window and is incredibly cost-efficient.

Model: gemini-1.5-flash
Pricing: Input: $0.075 / million tokens | Output: $0.30 / million tokens
Cost per request: ~$0.00015 (less than 1/50th of a cent)
Why choose it: Extremely high capability, massive context window (up to 1 million tokens), and the lowest pricing among tier-1 providers.
2. Groq (Paid Tier) — Fastest Speeds
Upgrading your free Groq key to a paid billing plan unlocks high-rate limits on Groq's custom LPU (Language Processing Unit) hardware.

Model: llama-3.3-70b-versatile (Large model)
Cost per request: ~$0.001 (1/10th of a cent)
Model: llama-3.1-8b-instant (Small model)
Cost per request: ~$0.00008 (1/100th of a cent)
Why choose it: Speeds are unbeatable (generates answers almost instantly in under 1 second).
3. OpenAI — Industry Standard
OpenAI's flagship lightweight model is highly accurate, fast, and fully supported out-of-the-box.

Model: gpt-4o-mini
Pricing: Input: $0.150 / million tokens | Output: $0.600 / million tokens
Cost per request: ~$0.0003 (1/30th of a cent)
Why choose it: Standard API compatibility, high reliability, and excellent overall coding and document understanding.

🚀 "One-Pass" Production Update Checklist
To safely push updates in production in one pass:

If updating UI/Styles: Update the script version tag in your index file (e.g. widget.js?v=2.0).
If updating Python code: Restart the server process.
If updating Prompts/RAG logic: Clear the SQL answer_cache table.
If updating Document indexing: Run rebuild_index.py.









How to increase FAQ Hits (Optimization Steps)
Audit Your Logs:
Go to the Admin Log Auditor page in your dashboard.
Look at what questions users are asking frequently (e.g., "What is the admission last date?" or "Hostel fees").
Add Them to the FAQ Database:
You can upload a simple CSV file containing common Questions & Answers through the Admin Panel's Ingestion Tab (select FAQ / Q&A Dataset as the source type).
Alternatively, these can be inserted directly into the PostgreSQL faq table.
The Result:
The next time a user asks one of those questions, the system will match it fuzzy-style in the database, return it instantly, increase your FAQ Hits counter, and lower your Avg Response time from 2012 ms!
